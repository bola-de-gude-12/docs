# Introducing the Treasury Module

The Treasury module simplifies financial oversight across multiple companies. It offers a consolidated view of payment-related data such as bank balances, open entries for vendors and customers, payment journals, proposed payments, and G/L entries. This centralized view makes it easier to control finances and provides a clear picture of liquidity across the entire group. 

In the Treasury setup, you can control the display of balances for companies by selecting either all companies or specific ones using filters. Similarly, for G/L accounts, bank accounts, customers, and vendors, you must choose whether to display balances for all entries, only open entries, or a filtered selection. 

Treasury Groups allow the consolidation of multiple Treasury companies into a single group for easier tracking and management, with permissions set to control access for specific users. Authorized users can view and manage the Treasury data for the groups they are assigned to, ensuring both data security and efficient oversight. For more information on Treasury Groups and how to set up the Treasure module, refer to the [Setting up the Treasury module](@CF-129) article.
