# Setting up line matching

As part of automatic matching, line matching is configured per template – so the matching rules you set up for a specific template apply to all documents for which that template is used.

To set up line matching:

1. Activate automatic matching for a selected template by following the guide under [Configuring automatic matching](../../../Continia%20Document%20Capture/Setting%20up%20Document%20Capture/Setting%20up%20order%20and%20receipt%20matching/@DC-68/).
2. In the template card of the selected template, on the **General** FastTab, enable the **Recognize Lines** setting. This makes a number of **Line Matching** fields available for you to configure on the **Purchase Documents** FastTab (see step 3 below).
3.  Under **Line Matching**, configure the following fields as needed.

> \[!IMPORTANT] When performing a partial match, the value under **Matched Quantity** on the **Invoice Matching** page might refer to the quantity based on the unit of measure in the document lines – and not the value on the order lines. This might result in a value with decimal parts on the posted purchase invoice, under **Quantity**. However, the actual value posted is always **Quantity (Base)**. To show or hide the standard quantity and unit cost fields for purchase orders, purchase return orders, posted purchase receipts, and posted return shipments, use the **Show Qty. and Unit Cost** or **Hide Qty. and Unit Cost** actions.

> \[!NOTE] As noted in the table above, you can allow for unit cost variance in two ways: expressed as a fixed amount or as a percentage of the recognized value. It's also possible to use both methods at the same time. If you do so, the smaller of the two applies.

> \[!TIP] For all of the matching criteria mentioned in the table above, Document Capture starts at the top of the lines listed in each document – and then select the first line whose criteria match the ones set up by you. Note that if you've only selected **Yes - always** or **Yes - if present** for one of the criteria, the first identified match may be incorrect. For example, if you've selected **Yes - always** for **Match Order No.** and Document Capture then identifies a potentially matching line in an invoice based on this one criterion, the identified line is indeed from the correct order, but it may not be the right match if the invoice contains multiple lines from the same order. Enabling additional matching criteria would most likely solve this issue. For this reason, it's generally recommended that you enable the three matching criteria **Match Order No.**, **Match Item No.**, and **Match Unit Cost**.

## Yes - always or Yes - if present?

When you set up line matching as described above, you can choose between the following three options for each of the matching criteria:

* **No**
* **Yes - always**
* **Yes - if present**

Selecting **No** for one of them leaves that matching criterion disabled, meaning that it's disregarded entirely in the matching process. By contrast, selecting one of the **Yes** options for a criterion includes it in the process in one of the following ways, depending on which option you choose:

* **Yes - always** - Document Capture uses any line value in the invoice/credit memo – even if it's empty – and that (empty) value must match perfectly with another document line's value. This means that if, for example, you choose this option under **Match Unit Cost** and the **Unit Cost** field of an invoice line is empty, Document Capture is forced to use this empty value, but it's unable to match it to anything.
* **Yes - if present** - Document Capture uses any line value that's present (that is, not empty) in the invoice/credit memo, and that value must match with another document line's value. If a line has an empty value, Document Capture disregards this and attempt to find matching lines based on the other matching criteria you've defined in the setup.

While **Yes - always** has the potential drawback of resulting in an unsuccessful match, it can also be a way to remind vendors that they must include certain specific details in their invoices and credit memos for the matching process to run smoothly. Whether to choose this option depends entirely on the needs and wants of your organization.
