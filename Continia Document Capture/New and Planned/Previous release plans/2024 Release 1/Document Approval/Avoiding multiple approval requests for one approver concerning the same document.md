# Avoiding Multiple Approval Requests for One Approver Concerning the Same Document

| Feature | Public preview | General availability online |
| --- | :-: | :-: |
| Avoiding multiple approval requests for one approver concerning the same document | {{checkmark}} Mar 2024 | {{checkmark}} Apr 2024 |

## Business value

With this feature, you'll avoid the hassle of ever having to approve the same document more than once. Having multiple approval tasks for the same document is not only redundant but also frustrating and unnecessarily time-consuming. The implementation of this feature means that you won't have to face such frustrations in the future.

## Feature details

The feature will ensure that no approver will have to approve the same document more than once, including if the approver is covering for a subordinate approver whose approval limit is insufficient.

For more information, see [Approval Sharing](@DC-37#avoiding-unnecessary-approval-requests).
