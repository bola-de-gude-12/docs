Continia continuously releases new features in accordance with the Feature Management program. This program allows features to be released as minor updates in between major updates, if necessary. There are still two major releases per year, in April and October.

Features in minor updates are optional until the next major release and then become mandatory with the major release. Any features released as part of a minor update are disabled by default, but administrators can choose to enable any or all of them from the **Continia Feature Management** page in Microsoft Dynamics 365 Business Central – which lists all available features for each app. The administrators can also safely disable the features again during the grace period leading up to the next major update, although this may not be possible for some of the features. If a feature can't be disabled for some reason, you'll be notified about this before enabling it. When the next major update is released, all minor-update features become mandatory and are enabled for all customers. At the same time, they disappear from the **Continia Feature Management** page.

Making new features available ahead of time enables you to test them and prepare your organization for the upcoming changes. Note that all minor updates released via Feature Management are listed in the table on the [Release plan for Continia {param1}](/Continia {param1}/New and Planned/Overview.md) page with a footnote stating that the features are available for preview.

> [!NOTE]
> There may not always be optional features available on the **Continia Feature Management** page, as their number varies from time to time. Immediately after any major release, the number of optional features available is likely to be zero.
