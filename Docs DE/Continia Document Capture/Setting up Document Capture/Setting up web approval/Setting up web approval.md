# Überblick über das Einrichten der Web-Genehmigung

Belege können entweder über den Microsoft Dynamics 365 Business Central-Webclient oder das Continia Web Approval Portal genehmigt werden. Das Web Approval Portal basiert auf der standardmäßigen Genehmigungsfunktionalität von Business Central. Es handelt sich jedoch um einen separaten Service, auf den lizenzierte Benutzer von [allen unterstützten Webbrowsern](/continia-document-capture/getting-started/frequently-asked-questions/minimum-requirements/additional-guidelines#browsers) aus zugreifen können.

> [!IMPORTANT]
> Obwohl Benutzer des Web Approval Portals keinerlei Zugriff auf Business Central benötigen, erhalten sie jedoch automatisch darauf Zugriff, sobald ihnen eine Lizenz für das Portal zugewiesen wurde (normalerweise eine Team Member-Lizenz). Das Web Approval Portal ist auf der Standardfunktionalität von Business Central aufgebaut, wodurch sich der Zugriff auf Business Central ergibt.

Das Web Approval Portal kann entweder von Continia (continiaonline.com) oder von Ihrem Unternehmen auf einem lokalen Server gehostet werden. Der Zugriff erfolgt in beiden Fällen über einen Webbrowser; der Unterschied besteht also nur beim Hosting. Welches Hosting für Sie in Frage kommt, hängt von Ihrer Business Central-Bereitstellung ab:

- **Wenn Sie Business Central Online nutzen**, können Sie nicht die lokal gehostete Version des Web Approval Portals verwenden; es kann nur die von Continia gehostete Version verwendet werden.
- **Wenn Sie Business Central On-Premises verwenden**, können Sie entweder die lokal gehostete Version oder die von Continia gehostete Version des Web Approval Portals verwenden.

Dadurch ergeben sich geringfügige Unterschiede bei der Einrichtung des Web Approval Portals, je nach Ihrer Gesamtkonfiguration. Weitere Informationen zum Einrichten des Web Approval Portals finden Sie unter den Links in der folgenden Tabelle:
