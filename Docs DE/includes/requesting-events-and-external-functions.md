Sie können die Hinzufügung von Event Publishern anfordern oder bestimmte Funktionen extern erstellen lassen, um Ihre Anpassungen zu unterstützen. Um das Risiko fehlerhafter Änderungen zu minimieren und reibungslosere Updates in Microsoft Dynamics 365 Business Central Online zu gewährleisten, sind die meisten Funktionen standardmäßig intern.

Prüfen Sie vor dem Absenden einer Anfrage, ob Ihr Bedarf bereits durch bestehende Event Publisher oder externe Funktionen abgedeckt ist. Verwenden Sie hierzu das [Event Recorder Tool in Business Central](https://learn.microsoft.com/en-us/dynamics365/business-central/dev-itpro/developer/devenv-events-discoverability), mit dem Sie alle während Ihrer Geschäftsprozesse veröffentlichten Ereignisse aufzeichnen und überprüfen können. Auf diese Weise können Sie verfügbare Ereignisse identifizieren, ohne den Code manuell durchsuchen zu müssen.

Wenn Ihre Anforderungen nicht enthalten sind, können Sie [eine Anfrage senden](/Continia {param1}/Getting Started/Help and Support.md). Wenn Ihre Anfrage angenommen wird, werden die Änderungen normalerweise in ein nächstes Service Pack aufgenommen (normalerweise innerhalb von zwei Monaten).

## So fordern Sie ein neues Event an

Wenn Sie ein neues Event anfordern, geben Sie folgende Details an:

- Name des Events
- Zweck des Events
- Genaue Stelle im Code, an der das Event hinzugefügt werden soll (einschließlich Screenshots oder Codeausschnitte)
- Erforderliche Parameter sowie eine Beschreibung jedes Parameters

Alle neuen Events folgen den Namenskonventionen von Microsoft.

## So fordern Sie eine externe Funktion an

So fordern Sie die externe Bereitstellung einer Funktion an:

1. Bestätigen Sie, dass die Funktion derzeit intern ist, indem Sie Ihre Erweiterung mit dem neuesten Continia {param1} Service Pack kompilieren oder den Quellcode überprüfen.
2. Wenn die Funktion nicht verfügbar ist, [erstellen Sie ein Supportticket](/Continia {param1}/Getting Started/Help and Support.md) (oder bitten Sie Ihren Partner darum). Die folgenden Informationen müssen angegeben werden:
   - Funktionsname
   - Objekttyp und -nummer
   - Begründung, warum die Funktion extern verfügbar gemacht werden soll
