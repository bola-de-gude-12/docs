# Selecting a remittance information template

When processing numerous payment documents, conveying all necessary details within the payment file's reference texts can be challenging. When using Continia Banking, default remittance information is automatically set up in the system and applied in the Banking Export Setup unless specified otherwise. The following describes how to change the Remittance Information template in different locations. If you want to learn more about adding your own custom remittance information template, refer to the [Adding a Remittance Information Template](@CB-117) article.

To change the Remittance Information settings in the Banking Export Setup:

1. Search ({{search}}) for and select **Banking Export Setup**.
2. Navigate to the **Remittance** FastTab to change the settings. For more information on the available fields, refer to the [Configuring Export Settings](@CB-23) article. 

To change the Remittance Information settings in a Payment Suggestion Template:

1. Search ({{search}}) for and select **Payment Suggestion Templates**.
2. Navigate to the **Remittance Information Template** column. For more information on Payment Suggestion Templates, refer to the [Setting up Payment Suggestion Templates](@CB-89) article.

To change the Remittance Information settings in the Payment Journal Setup:

1. Search ({{search}}) for and select **Payment Journal Setup**.
2. Navigate to the **Templates** FastTab, and select the template in the **Remittance Information** section. 

To change the Remittance Information settings on the Suggest Payment request page:

1. Search ({{search}}) for and select **Payment Journals**.
2. Go to the **Batch Name** field, select the three dots, and on the General Journal Batches page, select a journal.
3. On the action bar, select **Home** > **Suggest Vendor/Customer/Employee Payments**.
4. Navigate to the **Text Templates** section to find the **Remittance Information Template**. For more information on suggesting payments, refer to the [Suggesting and processing payments](@CB-109) article.
