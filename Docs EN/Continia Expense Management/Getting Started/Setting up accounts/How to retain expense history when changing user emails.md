# How to retain expense history when changing user emails
At Continia online, the unique identifier of a user is their email address. However, in Business Central a user's unique identifier is their Continia User ID. 

If the Continia user ID has not been changed and the **Keep History in Continia Online** setting in Expense Management is set for 6 or 12 months, use the following procedure to ensure the user's expense history is not lost.

To retain a user's expense history when changing their email:

1. Set all pending documents to **Open**.
1. Search {{search}} for **Continia Users**.
1. Update the email of the user(s) you want to export.
1. On the action bar, select **Export Users**.
1. Search {{search}} for **Expense Management Setup**.
1. On the action bar, select **Actions** > **Functions** > **Send History to Continia Online**.

Microsoft 365 users are able to log in as usual. However, users that log in through the portal must request a new password to do so.
