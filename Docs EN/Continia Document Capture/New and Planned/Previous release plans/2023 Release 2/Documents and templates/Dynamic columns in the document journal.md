# Dynamic Columns in the Document Journal

| Feature | Public preview | General availability online |
| --- | :-: | :-: |
| Dynamic columns in the document journal | - | {{checkmark}} Oct 2023 |

## Business value

By enabling you to add multiple customizable columns to the document list in the document journal, this feature will provide you with a wealth of flexibility for personalizing its appearance to suit your needs. You'll be able to add exactly the type of data you want to view, and it will be possible to both sort and filter this data in various ways.

## Feature details

The feature will enable you to customize your document journal list with a number of dynamic columns that can be sorted and filtered. A new action – **Set up Custom Columns** – will be added to the action bar of the document journal, and selecting this action will open the **Edit - Custom Field Card**, where you can then customize the list as needed.

You'll be able to add up to six columns to the list. These columns will have the following overall data characteristics:

* Two columns with date fields
* Two columns with text fields
* Two columns with number fields

The columns will be dynamic because you'll be able to choose the exact data to display for each column. You'll choose the data from a dropdown menu, and the data available here will be header template fields that match the data type of the column you're customizing. For example, if you want to add a date column to the document journal, the header template fields that are available in the dropdown menu will all be date fields. This will ensure that the columns can be filtered and sorted.

The data values in the dropdown menu will indicate what the header of the column will be. For instance, if you choose **DUEDATE** in the dropdown menu of a date column in the setup, the column header in the document journal will read **Due Date**.

For more information, see [Customizing List Columns in the Document Journal](@DC-156).
