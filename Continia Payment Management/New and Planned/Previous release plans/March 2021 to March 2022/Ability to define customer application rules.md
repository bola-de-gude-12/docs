#Ability to Set up Customer Application Rules

| Feature | General availability on-premises | General availability online | Public preview |
| --- | :-: | :-: | :-: |
| Ability to set up customer application rules | {{checkmark}} Sep 1, 2021 | {{checkmark}} Oct 1, 2021 |{{checkmark}} May 25, 2021|

##Feature details

With this feature you can define customer application rules, making it possible to manage how the payment text on a customer transaction can be used for the identification of the customer who created the payment. This will ease the process of applying customer payments in the cash receipt journal. For more information, see [Setting up customer application rules](@PM-174).

This feature is available from Payment Management version 3.0.0.0.
