# Sending Electronic Documents in Austria
In Austria, you can send electronic documents via email or through the PEPPOL eDelivery Network. Continia Document Output connects you directly to the PEPPOL eDelivery Network, eliminating the need for third-party integration. To learn more about connecting to the PEPPOL eDelivery Network, refer to the [Secure communication using Continia Delivery Network and PEPPOL](@DO-92) article.

All Austrian suppliers to the Austrian federal government are obliged to use e-invoices according to the Austrian ICT Consolidation Act. One way of complying with this is by using the PEPPOL eDelivery Network and the PEPPOL format. Other public sector entities (such as states, municipalities, and government-owned companies) are being integrated into the invoicing solution on a voluntary basis. The PEPPOL BIS 3 format is being adopted by businesses and public authorities in Austria, and Document Output supports this format.

> [!NOTE]
>
> Another format that has been used in Austria is ebInterface, but this is expected to eventually be replaced by PEPPOL. Document Output currently doesn't support ebInterface but may do so in the future.

You can also export and send electronic documents to customers in other countries. Most widespread formats are supported, but not all. For a list of supported formats, see [Supported Formats](@DO-57).

## Participant ID Austria

The following table lists the Participant IDs you can use if you want to participate in the Peppol network. It uniquely identifies the organization within the Peppol infrastructure and is used for routing electronic documents and messages.

| ---------------------------------------------------------- | ------ | ---------- |

| Firmenidentifikationsnummer der Statistik Austria          | AT:KUR |            |
| Österreichisches Verwaltungs bzw. Organisationskennzeichen | AT:GOV |            |
